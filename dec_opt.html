Decentralized Optimization

There has been much research done on minimizing functions when all information is available to a single agent. In the field of decentralized optimization, we are concerenced with minimizing function when information about the global function is spread out between many agents and communication is costly. There are many questions that arise in this scenario: Under what conditions do the agents agree, and how quickly? Under what conditions do they find the minimizer to the global function? What algorithms work best under what conditions? How do these properties change if each agent operates asynchronously? What happens when communication is sparsified and non-reliable. These are some of the questions I studied while at the NRL.

This problem is prvalent in modern-day machine learning where, for example, a large collection of images are stored on multiple machines in a data center for the purpose i=of image classification. There are also applications in sensor networks and robotic swarms, where the sensors and robots collect measurements in order to learn an underlying phenomenon, navigate an environment, or decide on an optimal control action. Some examples include Consensus based bundle algorithms and Pose graph optimization.

The Problem

To formalize the problem, in classical centralized optimization, we are often tasked with minimizing a function:

$$ \min_{x \in \mathbb{R}^p} F(x) $$

However, in our case, we have n agents that each have their own local functions $f_i(x)$ and the global function that we are trying to minimize is the collective sum over all these local functions:

$$ F(x) = {1\over n}\sum^n_{i=1}f_i(x)$$

We attempt to find the solution by solving the problem

$$ \min_{x_i \in \mathbb{R}^p} \sum^n_{i=1}f_i(x) $$ $$ \text{s.t. } x_i = x_j \; \forall \; i,j \in N_i $$

Where $N_i$ is the set of all neighbors for agent i. Agents can only communicate with agents in their neighbor hood, and at the end of the algorithm, all agents must agree with all their neighbors. Since we generally only look at graphs that are fully connected this implies that all agents agree on a solution. This problem is nontrivial since agents can only exchange state information, and must not transmit any details about their own local, private function.

Algorithms

There have been many algorithms that have been proposed to solve this problem in a distributed fashion. Througout my time at NRL, I studied and implimented many of these, some of which are listed below:

  • Decentralized Gradient Descent
  • AB Push Pull
    • Polyak Heavy-ball Momentum
    • Nesterov Acceleration
  • Decentralizeed Frank Wolfe
  • Fast Row Stochastic Optimization (FROST)
  • Distributed Nesterov Methods
  • Fast Row-Stochastic Optimization with Neseterov's Momentum (FROZEN)
  • Decentralized Projected Gradient
  • Distributed Successive Convex Approximation Algorithm over Time-Varying DigrAphs (SONATA)

MPI

One question of critical importance is "How effective are these algorithms when all agents act and communicate asynchronously?"" This is relevent because in real decentralized systems it is often infeasible to have all agents operate and communicate on a uniform "heartbeat". In order to simulate this asychronicity, we implimented our algorithms using pythons Message Passing Interface (MPI) package mpi4py. This allowed us to run each agent on it's own process asynchronously. We were then able to analyze the effects on convergence that differing levels of asychronicity produced.
